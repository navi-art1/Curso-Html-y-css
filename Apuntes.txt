Paginas Estaticas y dinamicas

    Páginas Estáticas

        Todos los productos para consumir, imagenes, informacion y no cambian , por ejemplo los blogspot
        Se les conoce como las paginas informativas.
        La informacion prácticamente no cambia.
        El cliente no va a poder generar algun tipo de cambio.

    Páginas Dinámicas

        Dejan de ser paginas y se convierten en aplicaciones, se utilizan bases de datos, para poder hacer login.
        Se puede empezar a interactuar.

    Conclusiones
        Normalmente podemos realizar una pagina dinamica para generar interaccion con un usuario.

        Fb si dejo 20min abierto, se actualiza todo y el contenido cambia, podemos postear algo, dema   s cosas
        por lo tanto es una pagina dinamica.

Etiquetas Multimedia
    Tipos de imagenes
    si utilizamos imagnes muy pesadas se volvera muy lenta la apertura de la pagina
        Lossy: Formato que nos permite perder calidad pero ayuda en el tiempo de carga
            - JPG/JPEG (Photographics Expets Group) se utiliza mas para fotografia, son enormes
                pero al comprimirla va a perder detalles, pero se va a renderizar muy rapido.
            - SVG (scalabe Vector Graphics) formato vector 
        Lossless: Formato sin perdida de calidad de la imagen, suelen ser pesadas
            como utilizan 256 colores, al momento de comprimir no se pierde mucho
            - Gif(Graphics Interchange Format) no se pueden comprimir
            - PNG8 (Portable Network Graphics)
            - PNG24 mas calidad

Optimizando imagenes
    No se deben utilizar imagenes de 3mg, xk va a tardar mucho tiempo en renderizar
    ¿Cual es el tamaño optimo para una imagen de proyecto?
        70KB - 100KB, es lo normal.
    - Utilizar 2 opciones: paginas en las cuales debemos subir las imagenes y nos retornará
        la imagen comprimida, las imagenes tienen metadatos
        * Tiny PNG (Tambien se puede colocar imagenes JPG)
        * Verexif
